# 🎣 PhishTank Simulator — Project README

> **An interactive cybersecurity awareness platform that teaches users to spot phishing emails and scam texts through gamified red-flag detection.**

---

## 📌 Project Overview

**PhishTank** is a browser-based interactive workshop / web app that exposes users to real-world examples of phishing emails and scam SMS messages. Players must identify and "catch" red flags — suspicious URLs, urgent language, spoofed sender addresses, grammar mistakes, and more — to earn points and level up their cyber-awareness skills.

The project is part of a broader **Digital Literacy & Cybercrime Awareness** portfolio spanning five tasks.

---


## 🎮 Task 1 — PhishTank Simulator (Presentation)

### What It Is
An interactive presentation introducing the **PhishTank** concept — a gamified phishing awareness simulator. The presentation covers:

- What phishing is and why it works
- How PhishTank teaches users to identify red flags
- Demo walkthrough of the web app interface
- Key statistics on phishing attacks in India and globally

### How the Simulator Works
1. User is shown a simulated phishing email or scam text
2. They click on elements they believe are red flags
3. Each correct identification earns points
4. A scorecard reveals missed red flags at the end of each round
5. Difficulty increases across levels (Basic → Intermediate → Advanced)

## 📧 Task 4 — Email Etiquette

### What It Is
A collection of professional email drafts and a comprehensive email etiquette checklist — produced as part of understanding how legitimate emails differ from phishing attempts.

### Sample Email Scenarios Covered
1. Requesting access to a system / resource
2. Reporting a suspected phishing email to IT
3. Sharing a cybersecurity awareness resource with a team
4. Following up on an unanswered security query
5. Declining a suspicious vendor solicitation politely

### Etiquette Checklist Preview *(full list in `etiquette_checklist.md`)*
- [ ] Clear, specific subject line (no vague "Hi" or "FYI")
- [ ] Professional greeting and sign-off
- [ ] No urgent / threatening language
- [ ] Verified sender domain before replying
- [ ] No sensitive data in plain-text body
- [ ] Attachments are expected and scanned before opening
- [ ] Reply-to address matches the sender domain

## 🎯 Learning Objectives

By completing this project, participants will be able to:

1. **Identify** common phishing tactics — urgency, spoofed domains, lookalike URLs
2. **Recognise** the anatomy of a legitimate vs. malicious email
3. **Apply** email etiquette best practices in professional communication
4. **Understand** real-world cybercrime case studies and their business impact
5. **Implement** personal and organisational cybersecurity prevention measures
